# Codex Guide

Principles

1. Respect the architecture.
2. Never redesign core abstractions.
3. Implement one module per task.
4. Prefer small, reviewable commits.
5. Keep the Execution Engine UI-agnostic.
6. Keep visual components stateless where possible.
7. Do not bypass the Execution Trace.

Every implementation task should begin by identifying:
- Inputs
- Outputs
- Interfaces
- Tests
